Making a Database Handler pt.4 - GitHub

Gabriel Menezes | Wed, Jun 24, 20

Last update: Wed, Jun 24, 20

In this tutorial, I’ll show you how to add an existing project to GitHub! And why is this important? So we keep track of the changes in our code, if we make any mistake we can go back for the previous version and also we can open our project to colaboration.

Previous episode - Making a Database Handler pt.3 - Psycopg

GitHub, what is GitHub? Where does it lives? What does it eats?

GitHub is a repository hosting service for colaboration and version control using Git, with GitHub is easier to people anywhere in the world work together in projects.

GitHub offers professional and enterprise accounts but also offers plans free of charge.

But what is Git?

It’s an OpenSource project started by Linux creator Linus Torvalds with the purpose, like other version control systems, to manage and store revision projects. Git could be used to manage any type of files, such as word documents, but is mostly used to store code. Think of it like a filing system for any draft or document.

As an conclusion we chan think of GitHub as a Git repository hosting service.

Creating a GitHub repository

Go to GitHub to creat your account clicking on SignIn button, after that just go to your main page that is the same as the previous GitHub link you’ve clicked previously and click on New aside of Repositories like the image below.

Or at the top right of navbar.

  1. Choose an smart & cool name
  2. Describe the main idea
  3. Choose if you want to be Public or a Private repository
  4. You may skip the step where does it say at the picture below, but I’ll not, and I’ll show you how to do that.
    • .gitignore: It’s a list of files and folders that should not be send from your local repository to github, like __pycache__/ or production keys for example.

And then the created repo

Initialize a Git repository

  • init: Create an empty git repository or reinitialize an existing one

Ouptput:

Initialized empty Git repository in /home/gmenezes/Documentos/repos/python-database-handler/.git/

Add the files

  • add: Add file contents to the index

you can check the result with the command below

  • status: Show the working tree status

Output:

	new file:   Dockerfile
	new file:   Makefile
	new file:   Pipfile
	new file:   Pipfile.lock
	new file:   database_handler.py
	new file:   databases_access.json
	new file:   init.sql
	new file:   query.sql
	new file:   script.py

Link your local repository to GitHub repository

  • remote add origin: Add the GitHub repository as origin

Get the GitHub files (.gitignore, LICENCE and README.md)

  • pull: Fetch from and merge with another repository or a local branch
  • --prune: Clean outdated branches

Commit the staged files

  • commit: Record changes to the repository
  • -m: Use the given as the commit message.

Finally push the files to GitHub

  • push: Update remote repository with local objects

Now Check your repository on GitHub and you shall see all your files stored.
